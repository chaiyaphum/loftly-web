ฟีเจอร์หลัก

🔍 Merchant Lookup — ใช้บัตรไหนคุ้มที่ร้านไหน

พิมพ์ชื่อร้าน (Starbucks · Grab · Shopee · ฯลฯ) → ดูว่าใช้บัตรใบไหนคุ้มที่สุดตอนนี้ พร้อมโปรโมชัน active + ตัวเลข THB ที่ประหยัดได้ — ไม่ต้องกรอก spend ล่วงหน้า

ลอง /merchants · /merchants/starbucks · /merchants/grab

📊 Card Selector — จัดอันดับบัตร 3 ใบที่คุ้มสุด

กรอกข้อมูล spend + เป้าหมาย (ไมล์ · เงินคืน · สิทธิประโยชน์) 1 ครั้ง → ได้คำแนะนำ 3 ใบ พร้อมเหตุผลภาษาไทยและตัวเลข THB ที่ประหยัดได้ต่อเดือน

ลอง /selector

🔥 Promos Today — ฟีดโปรโมชันบัตรเครดิตไทยสด

รวมทุกโปรที่ active วันนี้ จาก KTC · SCB · KBank อัปเดตทุกวัน จัดกลุ่มตามร้าน พร้อมนับถอยหลังวันหมดอายุ (≤ 7 วัน = แดง, ≤ 21 วัน = ส้ม)

ลอง /promos-today

💎 Valuation Engine — มูลค่า THB ต่อแต้ม

มูลค่า 8 สกุลแต้ม (KrisFlyer · Asia Miles · Royal Orchid · Bonvoy · ฯลฯ) พร้อม confidence band (exact / directional / range ตาม VALUATION_METHOD.md) และ history 4 สัปดาห์ล่าสุด

ลอง /valuations · /valuations/krisflyer

📖 Card Reviews — รีวิวรายใบแบบตรงไปตรงมา

รีวิวบัตรเครดิตไทย เน้นอัตราแลกไมล์ ค่าธรรมเนียม สิทธิประโยชน์จริง ไม่มีโฆษณาแฝง · ทุกลิงก์ Apply มีป้าย Affiliate disclosure ชัดเจน

ข้อจำกัดที่ควรรู้

เวอร์ชันทดสอบ (staging · dev-only phase) — ยังไม่เปิดใช้งานจริง มีข้อจำกัดที่ควรรู้:
  • ระบบอีเมลยัง 🚫 — magic-link + welcome email ยังเขียนเป็น log เท่านั้น (Resend ยังไม่ได้ provision ตาม dev-only phase memo) · ล็อกอินได้เฉพาะผ่าน OAuth (Google/Apple/LINE) หรือใช้ guest session
  • Sentry error capture ยัง 🚫 — errors ไม่ถูก forward; ดูจาก DO runtime logs แทน
  • LLM provider — LOFTLY_LLM_PROVIDER อาจยังเป็น deterministic บน staging · Card Selector + Valuation + Chat ใช้ rule-based fallback (ไม่ใช่ Claude จริง) จนกว่าจะ flip เป็น anthropic (env-var change บน loftly-api-staging · ไม่ใช่ code change)
  • Feature flags — MERCHANTS_REVERSE_LOOKUP_ENABLED + LOFTLY_FF_SELECTOR_PROMO_CONTEXT : ถ้ายัง off · /merchants/* จะคืน 404/503 และ promo chips บน Selector results จะไม่แสดง
  • LivePromoStrip — แถบบน landing จะหายไปอัตโนมัติถ้า deal-harvester sync stale > 72 ชม. (brief §15.5 · ไม่ fake liveness)
  • /promos-today อาจว่าง — ถ้า backend /v1/promos?active=true ยังไม่ expose หน้านี้จะ render empty-state + link ไป /merchants
  • Filter UI บน /cards — ยังเป็น grid เปล่า · ยังไม่มี filter ตาม issuer/tier/network (deferred)
  • Chat panel บน Selector results — UI scaffold อยู่แล้ว แต่ยังไม่ต่อ Haiku API จริง (POST_V1 §1 future)
  • Hero illustrations — ตอนนี้เป็น CSS placeholder เรียบ ๆ · รอ designer ทำ custom art
  • Dark mode — token ทั้งหมด prepared แล้ว (CSS variable swap ready) แต่ skin ยังไม่ active · v1.1
  • CI / Cloudflare Pages — deploy.yml + CI checks ใน loftly-web fail เพราะ GitHub Actions billing issue · ไม่กระทบ DO staging (DO App Platform ดึง git โดยตรง · ไม่ผ่าน Actions runner)
